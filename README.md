## Архитектура решения
Решение представляет собой **гибридный поисковый пайплайн**, объединяющий лексический поиск (BM25) и семантический поиск (векторные эмбеддинги) с Min-Max нормализацией и линейной комбинацией скоров.

---

## 1. Обработка HTML статей

Для очистки HTML-разметки используется библиотека **BeautifulSoup** с методом `get_text(separator=" ", strip=True)`. Дополнительно удаляются теги `<script>`, `<style>`, `<nav>`, `<footer>`, которые содержат служебный код и не несут смысловой нагрузки.

После очистки текст нормализуется:
- Приведение к нижнему регистру
- Удаление спецсимволов (оставляются только буквы, цифры и пробелы)
- Ограничение длины до 1500 символов для векторной модели

---

## 2. Использованные признаки, модели и алгоритмы

### Лексическая ветка (BM25)
- **Алгоритм:** BM25Okapi из библиотеки `rank_bm25`
- **Параметры:** `k1=4.0`, `b=0.5`
- **Токенизация:**
  - Лемматизация через `pymorphy3` (приведение слов к начальной форме)
  - Генерация биграмм для захвата устойчивых выражений (например, "авито_доставка", "возврат_денег")
  - Удаление стоп-слов (приветствия, местоимения, служебные части речи)
- **Индексация:** Объединение токенов заголовка и тела статьи

### Семантическая ветка (векторный поиск)
- **Модель:** `intfloat/multilingual-e5-large` (`560M параметров)
- **Префиксы:** `passage:` для статей, `query:` для запросов (стандарт E5-моделей)
- **Нормализация:** L2-нормализация эмбеддингов для косинусного сходства

### Гибридизация
- **Метод:** Min-Max нормализация скоров каждой ветки в диапазон [0, 1]
- **Формула:** `hybrid_score = 0.5 * vector_score + 0.5 * bm25_score`
- **Параметр alpha:** 0.5 (равный вес обеих веток, подобран через Grid Search)

## 3. Проверка качества на calibration.f

Качество оценивалось по метрике **MAP@10** (Mean Average Precision на топ-10 позициях).

### Процесс валидации:
1. Построение индексов BM25 и векторных эмбеддингов на всей базе статей
2. Для каждого из 500 запросов из `calibration.f`:
   - Расчет скоров BM25 и векторного сходства
   - Гибридизация и сортировка
   - Извлечение топ-10 статей
   - Сравнение с `ground_truth`
3. Подсчет AP@10 для каждого запроса и усреднение

### Grid Search по параметрам:
- Перебор `alpha` от 0.2 до 0.8 с шагом 0.1
- Оптимальное значение: **alpha=0.5** (MAP@10 = 0.4296)

## 4. Анализ ошибок и борьба с ними

### Тип 1: Морфологические расхождения
**Пример:** Запрос "оплачиваешь", статья "оплата"
**Решение:** Лемматизация через `pymorphy3` приводит все формы к начальной

### Тип 2: Сленг и разговорные выражения
**Пример:** "приедут деньги", "наличка", "бабки"
**Решение:** Векторная модель понимает семантическую близость, BM25 компенсируется биграммами

### Тип 3: HTML-шум
**Пример:** Служебные кнопки, навигация, футеры
**Решение:** BeautifulSoup удаляет теги `<script>`, `<style>`, `<nav>`, `<footer>`

## 5. Используемые технологии

- `rank_bm25==0.2.2` — алгоритм BM25
- `sentence-transformers==5.6.0` — векторные эмбеддинги
- `pymorphy3==2.0.6` — морфологический анализ
- `beautifulsoup4==4.14.2` — очистка HTML
- `torch==2.6.0+cu124` — фреймворк для нейросетей
- `numpy==2.2.6`, `pandas==2.3.3` — обработка данных

Все модели — open-source, запускаются локально, без внешних API.

## 6. Код решения и инструкция по запуску

### Структура проекта:
```text
.
├── main.py                  # Основной скрипт генерации answer.csv
├── requirements.txt         # Зависимости
├── README.md                # Данное описание
└── answer.csv               # Финальный ответ
```

### Инструкция по запуску:
```bash
# Установка зависимостей
pip install -r requirements.txt

# Генерация ответа
python main.py
```

Скрипт автоматически загрузит модели, построит индексы и создаст файл `answer.csv` с предсказаниями для тестовой выборки. Код полностью воспроизводим благодаря фиксированным сидам для `random`, `numpy`, `torch` и детерминированным операциям CUDA.
